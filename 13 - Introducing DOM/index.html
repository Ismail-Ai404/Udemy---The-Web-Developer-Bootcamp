Understanding the DOM in JavaScript

How your browser sees and allows you to control the webpage through JavaScript objects

üåê What is the DOM?

The Document Object Model (DOM) is a browser-provided interface that represents your HTML page as a structured tree of JavaScript objects. Every element‚Äîtags, text, attributes‚Äîis accessible and modifiable using JavaScript.

üìÑ Everything is an Object

When you load a webpage, the browser parses your HTML and turns it into a live object structure in memory. You can inspect this using the developer tools console.

Try running this command:

console.dir(document);

This command prints the entire document object with its properties and methods. You‚Äôll find:

  • document.head ‚Äì the <head> section
  • document.body ‚Äì the <body> content
  • document.links, images, scripts, etc.

üîç Exploring the DOM

You can narrow your exploration to parts of the page like this: 

console.dir(document.body);

And modify them dynamically:

document.body.style.backgroundColor = "black";
document.body.innerHTML = "<h1>Hello from JavaScript!</h1>";

These lines change the page's appearance on the fly. That‚Äôs the power of the DOM!

üß† Why It Matters

Mastering the DOM is essential for:

  • Reading or updating HTML content
  • Reacting to user actions (clicks, input, etc.)
  • Adding or removing elements dynamically
  • Creating smooth interactive user experiences
üí° Tip: Open your browser‚Äôs developer tools and run console.dir(document) to start exploring the live DOM.

üß± Selecting Elements with document.getElementById()

When you write HTML and CSS, you‚Äôre describing how a page should look ‚Äî but the browser transforms that into a living object model using JavaScript. These objects form a structured tree that you can interact with.

One of the simplest ways to get an element is using:

document.getElementById("myElement");

This selects the element with the ID myElement. Let's say your HTML contains: 

<div id="myElement">Hello DOM</div>

You can grab this element in JavaScript:

const a = document.getElementById("myElement");

Now a holds a reference to the real DOM object! You can inspect it using: 

console.dir(a);

This will display all the properties and methods of the element ‚Äî just like with document. You‚Äôll see things like:

  • innerText ‚Äì the text inside the element
  • style ‚Äì an object representing all the inline styles
  • children ‚Äì any nested DOM nodes

üå≥ Understanding the DOM Tree Structure

The DOM is hierarchical ‚Äî think of it like a tree. At the top is the document object, and beneath it lie its children:

Example:

This would show the first element inside the body ‚Äî maybe a <header>, <div>, etc.

üì∏ Selecting Elements with getElementsByTagName()

If you want to grab all elements of a certain tag ‚Äî like all <img>, <p>, or <section> tags ‚Äî you can use:

This returns an HTMLCollection containing all matching elements. Example:

üí° What exactly is returned?

getElementsByTagName() returns an HTMLCollection, not an array. You can loop through it using a traditional for loop or for...of, but not with array methods like map() or forEach() (unless converted). 

// Won't work:

// Will work:

  console.log(img.src);

üìö What are these things inside?

Each item in the collection is an Element ‚Äî a DOM object with many properties and methods:

  • src ‚Äì source of an image
  • width, height
  • style ‚Äì allows dynamic styling
  • classList ‚Äì for adding/removing classes
  • setAttribute(), removeAttribute(), etc.

We can change their size like this:

  img.style.width = "300px";
}

üåê Common tag names you can select: div, img, p, a, ul, li, h1, section, article, etc.

üìÇ The Class Collector getElementsByClassName()

When you're looking for all elements that wear the same uniform (i.e., share a CSS class), this is your go-to method.

This returns an HTMLCollection of all the elements that have the class cool.

“Hey browser, give me everyone wearing the ‚Äòcool‚Äô T-shirt.”

üß† Things to know:

  • Returns a collection, not a single item.
  • You can loop over it with for...of, but not directly with .map().
  • The result is live ‚Äî it updates when the DOM changes.

console.log(allBoxes.length);

üìå Manipulating Text with innerText, textContent, and innerHTML

When it comes to grabbing or updating the visible content inside elements, you've got three amigos:

  • innerText
  • textContent
  • innerHTML

üßê The Key Differences

<div id="hidden-text" style="display: none;">
  This won't show up with innerText.
</div>

const div = document.querySelector('#hidden-text');

console.log(div.innerText);     // "" (ignores hidden text)
console.log(div.textContent);  // "This won't show up with innerText."
console.log(div.innerHTML);    // "This won't show up with innerText."

- innerText: Gets or sets the visible text (ignores elements hidden with display: none).
- textContent: Gets all the text, even if it's hidden.
- innerHTML: Gets or sets the entire HTML inside an element.

‚úèÔ∏è Editing with innerHTML

const div = document.querySelector('#my-box');

// Add bold HTML directly
div.innerHTML = "<strong>Hello world!</strong>";

innerText will just insert the text as-is, not render any HTML:

div.innerText = "<strong>Hello world!</strong>"; 
// This will literally show <strong>Hello world!</strong>

‚ûï Appending with innerHTML +=

You can even use += to add more HTML inside the element:

const list = document.querySelector('ul');
list.innerHTML += "<li>New Item</li>";

üìç Selecting Elements to Manipulate

Use document.querySelectorAll() to grab all matching elements and update them:

const boxes = document.querySelectorAll('.card');

boxes.forEach(card => {
  card.innerText += " ‚ú® Updated!";
});

‚úÖ Summary

  • innerText ignores hidden elements
  • textContent includes all text (even hidden)
  • innerHTML reads/writes HTML, and lets you use += to add more
  • Great power = great responsibility (don't mess up your DOM!)

Changing Styles with JavaScript

When you access styles via JavaScript, only the inline styles are directly accessible through the .style property. Styles defined in external CSS files or internal <style> tags aren't directly available this way due to possible multiple files and specificity issues.

To get the computed styles applied to an element, use:

const h1 = document.querySelector('h1');
const computedStyles = window.getComputedStyle(h1);
console.log(computedStyles.fontSize);

This allows you to inspect the actual final style values as rendered in the browser, regardless of whether they came from CSS files, inline styles, or elsewhere.

You can still manipulate styles directly by setting properties on the element's style object. Remember: JavaScript uses camelCase instead of kebab-case. For example:

const h1 = document.querySelector('h1');
h1.style.fontSize = '3rem'; // NOT h1.style.font-size

This lets you override the CSS directly via JavaScript, applying changes instantly on the page.

Note: In CSS, properties like font-size are written in kebab-case, but in JavaScript's DOM style manipulation, they must be written in camelCase like fontSize.

Using classList (Recommended Way)

When you're manipulating classes, it's better to use element.classList instead of getAttribute and setAttribute. Here's why:

// BAD WAY: overwrites all other classes
const h2 = document.querySelector('h2');
const currentClasses = h2.getAttribute('class');
h2.setAttribute('class', `${currentClasses} purple`);

// RECOMMENDED WAY:
h2.classList.add('purple');     // Adds a class
h2.classList.remove('text-dark'); // Removes a class
h2.classList.toggle('visible');   // Toggles class presence
h2.classList.contains('fs-4');    // Checks for a class

Why use classList? Because setAttribute('class') replaces the entire class list. That means if you just want to add a class, you'll need to first get the current class string, concatenate it, and set it again ‚Äî a hassle and error-prone. classList makes it easier, cleaner, and safer.
